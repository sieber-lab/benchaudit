## Repository Purpose
BenchAudit is a reproducible audit framework for molecular property and drug-target interaction (DTI) benchmarks. It standardizes molecular representations, quantifies split hygiene and similarity, identifies label inconsistencies and activity cliffs, and optionally trains baseline predictive models for sanity checking.

## Pipeline Structure
1. Configuration ingestion: `run.py` reads one or more YAML benchmark configs.
2. Data loading: `utils/loader.py` normalizes split data into canonical tables (`train`, `valid`, `test`) with consistent column names.
3. Data auditing: `utils/analysis.py` computes contamination, similarity, label conflicts, and (for DTI) sequence/structure alignment diagnostics.
4. Optional baseline benchmarking: `utils/baselines.py` trains models on train split features and evaluates on test split labels.
5. Artifact persistence: `utils/__init__.py` (`ResultWriter`) serializes summary and drill-down outputs to disk.

## Expected Inputs
| Input | Format | Scientific Role |
| --- | --- | --- |
| Benchmark configuration | YAML (`configs/*.yml`) | Declares benchmark type, task (`classification` or `regression`), split source paths/API metadata, and preprocessing parameters. |
| Molecular records | CSV/TSV/Parquet or API-provided tables | Provides ligands (SMILES) and labels for train/valid/test auditing and model evaluation. |
| DTI target information (optional) | Sequence and target-ID columns | Enables cross-split target leakage checks and sequence-level redundancy/alignment analysis. |
| Foldseek metadata/alignment files (optional) | JSONL and `.m8` | Enables structure-aware target similarity diagnostics when available. |

## Produced Outputs
| Output Artifact | Format | Interpretation |
| --- | --- | --- |
| `summary.json` | JSON | Compact benchmark-level statistics: split sizes, contamination, similarity summaries, conflict/cliff counts, and DTI-specific diagnostics. |
| `records.csv` | CSV | Row-level canonical audit table with cleaned structures, labels, split tags, and optional target annotations. |
| `conflicts.jsonl` | JSONL | Detailed records of inconsistent labels among duplicate or comparable entities. |
| `cliffs.jsonl` | JSONL | Detailed activity-cliff pairs with high similarity but discordant labels. |
| `sequence_alignments.jsonl` | JSONL | DTI sequence alignment diagnostics for cross-split target overlap. |
| `structure_alignments.jsonl` | JSONL | DTI structure-level alignment diagnostics when Foldseek inputs exist. |
| `performance.json` | JSON | Baseline model predictions and evaluation metrics (when benchmarking is enabled). |

## Operational Interface
- Primary CLI: `python run.py --config <path.yml> --out-root runs [--benchmark]`
- Batch mode: `python run.py --configs configs --out-root runs [--benchmark]`
- Scientific interpretation: the audit path (`--benchmark` omitted) quantifies dataset hygiene; the benchmark path (`--benchmark`) adds model-based performance context.
